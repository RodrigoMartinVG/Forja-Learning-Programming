# Separar source, código y proceso

Para cada caso, elegí una sola categoría entre `source`, `código cargado`, `dato` o `programa en ejecución`.

> Simulador: apoyate en el editor, el botón de cargar y el historial. Este ejercicio está pensado para que esa separación entre texto editable, memoria ya cargada y ejecución quede visible, no sólo nombrada.

## Consigna

| Caso | Categoría |
|---|---|
| el archivo del repo donde una persona edita el programa | `?` |
| la instrucción `101: ADD r0, [41]` ya ubicada en memoria | `?` |
| el valor `5` guardado en `mem[41]` | `?` |
| el mismo código con `pc=102` y `r0=12` | `?` |
| una secuencia de instrucciones todavía escrita para lectura humana en un archivo `.c` o `.rs` | `?` |
| `mem[42]=12` como resultado de `STORE` | `?` |

La idea es separar planos, no describir cómo compila el programa.
